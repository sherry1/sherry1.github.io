node.js学习笔记（一）

  • nodejs

认识node

  • 是什么？
    不是一种独立的语言，不是一个js框架，不是浏览器端的库
    她是一个让js运行在服务端的开发平台，它跳过了Http服务器，直接面向前端开发

  • 能做什么?
    她内建了http服务器支持，所以可以开发个网站，应用等
    她还可以部署到非网络应用的环境下，比如一个命令行工具等、
    可以调用c/c++,可以充分利用已有的诸多数据库，也可以将对性能要求非常高的部分用C/C++来实现

  • 有何特点?
    异步式I/O
    事件驱动

  • 性能如何?
    通过以下方式提升性能
    异步式I/O和事件驱动代替多线程
    使用v8作为javascript引擎
    使用高效的libev和libeio库支持事件驱动和异步式I/O

node.js编程

一、hello world的实现
方法一：编辑console.log('hello world');，保存为helloworld.js文件，打开终端，进入helloworld.js所在的目录，执行以下命里node helloworld.js
方法二：$ node -e "console.log('Hello World');"
方法三：使用node的REPL模式（REPL模式即输入-求值-输出循环），打开shell，输入node就进入了REPL模式，会出现一个”>”提示符提示你输入命令，直接输入”console.log(‘Hello World’);”`。

二、建立HTTP服务器
node.js与php不用的是她不需要预先配置强大而复杂的http服务器，譬如Apache,她将‘http服务器’这一层抽离，直接面向浏览器用户。

//app.js
var http = require('http');
http.createServer(function(req, res) {
res.writeHead(200, {'Content-Type': 'text/html'});
res.write('<h1>Node.js</h1>');
res.end('<p>Hello World</p>');
}).listen(3000);
console.log("HTTP server is listening at port 3000.");

运行 node app.js 命令，打开浏览器访问http://127.0.0.1:3000,即可看到效果
能用node.js实现http服务器最主要的一点就是她提供了http模块，通过调用此模块来实现。

tips：
修改nodejs的代码保存后不会立即生效，必须终止再nodejs再重新运行方可生效，因为nodejs只有在第一次引用时才回去解析脚本文件，之后会直接访问内存，避免重复载入，这样有利于提高性能，但不利于调试，所以我们使用supervisor来监视代码的改动，并自动重启nodejs，使用方法如下：

$ npm install -g supervisor  //安装
....

$ supervisor app.js          //启动

三、异步式I/O与事件式编程
1、同步式I/O（阻塞式I/O）
线程——遇到I/O操作（耗时）——操作系统剥夺其CPU控制权——线程暂停执行——资源让给其他线程（所以该模式必须是多线程）——I/O操作完毕——恢复该线程的CPU控制权，令其继续执行
2、异步式I/O（非阻塞式I/O）
线程——遇到I/O操作（耗时）——将I/O请求发送操作系统——线程继续执行
操作系统完成I/O操作——通知线程（以事件的形式）——线程在特定时候处理这个事件（线程必须事件循环，不断检查有无未处理的事件）

四、事件循环机制
nodejs始终在事件循环中，但没有显式的事件循环函数，由libev库实现

五、模块和包（支柱）
传统js通过script标签来实现脚本模块的拆分和组合
nodejs提供require函数来调用模块，而且模块都是基于文件的，模块和包没有本质区别，包可以理解成实现某个功能模块的集合。
1、什么是模块
nodejs应用程序的基本组成部分，文件与模块一一对应，这个文件可以是js代码，json或者编译过的C/C++扩展。例如之前例子中的var http = require('http')，其中的http就是nodejs的核心模块，内部用C++实现，外部用js封装，我通过require函数获取使用其中对象。
2、创建及加载模块

  • 创建模块
    就是创建文件，这很简单，我们主要关注的是获取
    nodejs提供了exports和require两个对象，其中exports是模块公开的接口，require用于从外部获取模块接口（对象）

  • 单次加载
    require不会重复加载模块，即无论调用多少次require，获取的模块都是同一个

  • 覆盖exports

//hello.js
function Hello() {
var name;
this.setName = function(thyName) {
name = thyName;
};
this.sayHello = function() {
console.log('Hello ' + name);
};
};
exports.Hello = Hello;

此接口对象是export

//hello.js
function Hello() {
var name;
this.setName = function(thyName) {
name = thyName;
};
this.sayHello = function() {
console.log('Hello ' + name);
};
};
module.exports = Hello;

此接口对象是Hello,是用Hello覆盖了exports
不能用exports直接赋值代替对module.exports赋值，exports和module.exports是指向同一对象的变量，模块执行完毕后它本身会释放，而module不会

3、创建包
我们可以通过require一个文件夹（即包），来获取文件夹中所有文件输出的对象
方法一：创建个文件夹somepackage,在其中创建index.js
方法二：创建个文件夹somepackage，在其中创建个package.json的文件，内容如下{ "main" : "./lib/interface.js" },然后将index.js重命名为interface.js并放入lib子文件夹下
可以在somepackage目录下运行npm init，生成符合标准的package.json
